## Installation

For the OSS version, the v9 release is available in the [Maven Central repository](https://mvnrepository.com/artifact/org.jobrunr/jobrunr).

For Pro users, the v9 release is available on our private Maven repository.

Simply replace the version you’re on with `9.0.0`.

## Micronaut 5 support

As of v9, JobRunr OSS and JobRunr Pro require Micronaut 5. Before upgrading to JobRunr v9, you must upgrade your application from Micronaut 4 to Micronaut 5.

Support for Micronaut 4 has been removed and is no longer available in JobRunr v9.

## EventBus
{{< badge version="professional" >}}JobRunr Pro{{< /badge >}}

JobRunr Pro now includes a built-in EventBus that supports multiple inter-server communication channels, such as UDP multicast and PostgreSQL LISTEN/NOTIFY.

With the EventBus, job enqueue events are propagated to all relevant servers immediately, allowing enqueued jobs to start executing almost instantly instead of waiting for the next polling cycle.

### Usage

<!-- TODO mention auto configuration  -->

In order to use the event bus you will need to either provide a `MulticastEventTransport` bean if you were previously 
using `jobrunr.multicast-group-address` or configure one using `org.jobrunr.configuration.JobRunrConfiguration#useEventTransport`
if you were previously using `org.jobrunr.configuration.JobRunrConfiguration#useMulticastAddress`. 
You are able to configure this using either a `PostgresEventTransport` if you are using PostgreSQL or a `MulticastEventTransport`.

If you do not explicitly configure an EventTransport the default event transport used is the `MulticastEventTransport`.

<!-- TODO use codetabs -->

```java
JobRunrPro
    .configure()
    .useEventTransport(new PostgresEventTransport(storageProvider))
```

> [!IMPORTANT]
> When you use the `MulticastEventTransport` like with the previous implementation of multicast please ensure that your network supports multicast.

## Pausing of Batch Jobs
{{< badge version="professional" >}}JobRunr Pro{{< /badge >}}

The pausing batch job feature allows you to pause all non processing child jobs of a batch job executing. Once you pause
a batch job any child job that was in enqueued, scheduled or pending will be paused and not execute until you resume the batch job.

> **Important:** You can only pause a batch job once all the child jobs have already been created

### Usage
In order to pause a batch job you can either press the pause and resume buttons directly in the dashboard on a given
batch job, or you can call our api endpoints `/jobs/{batchJobId}/pause` and `/jobs/{batchJobId}/resume` to pause and resume
a batch job respectively. 

Pausing a batch job once all the child jobs have moved to the processing stage will still complete the batch job as
none of those child jobs will be paused.

## Breaking Changes

### API Changes
We removed a number of API's that were deprecated in v8:
- The deprecated `JobRunrConfiguration#useMicroMeter` has been removed, use `JobRunrConfiguration#useMetrics` instead 
- The deprecated `JobBuilder#withDetails` has been removed, use `JobBuilder#withJobLambda` instead
- The deprecated `RecurringJobBuilder#withDetails` has been removed, use `RecurringJobBuilder#withJobLambda` instead
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} The deprecated `JobSearchRequestBuilder#withQueue` has been removd, use `JobSearchRequestBuilder#withPriorityQueue` instead
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} The deprecated `RecurringJobSearchRequestBuilder#withQueue` has been removd, use `RecurringJobSearchRequestBuilder#withPriorityQueue` instead
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} The deprecated `JobServerFilter#getProgress` has been removed, use `JobServerFilter#getProgressAsRatio` instead

### Micronaut
- Dropped support for Micronaut 4, use Micronaut 5 instead

### Kotlin Exposed
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} Kotlin Exposed now requires v1 

### Multicast {{< badge version="professional" >}}JobRunr Pro{{< /badge >}}
- Method `org.jobrunr.configuration.JobRunrConfiguration#useMulticastAddress` has been dropped, you'll need to configure a `MulticastEventTransport` via `org.jobrunr.configuration.JobRunrConfiguration#useEventTransport`.
- Property `jobrunr.multicast-group-address` has been dropped, you'll need to provide configure `MulticastEventTransport` bean.
- `org.jobrunr.server.BackgroundJobServer#publishJobEnqueuedMessage` has been dropped you may use `org.jobrunr.server.BackgroundJobServer#publishEvent` instead.
- `BackgroundJobServer` has constructor changes to accept an `EventTransport`
- `JobRunrDashboardWebServer` has constructor changes to accept an `EventTransport`
- `DashboardApiHandler` and `DashboardDbDataProvider` have constructor changes to accept an `EventBus`
- `JobScheduler` and `JobRequestScheduler` have constructor changes to accept an `EventTransport`.
- `StorageProviderClusterConfiguration` has constructor changes to accept an `EventBus` instead of multi-cast-group-address.
- `JobEnqueuedMessagePublisher` has removed, you may implement an `EventTransport` instead.
- `MulticastMessagePublisher` and `MulticastMessageReceiver` have been removed, see `MulticastEventTransport` instead.

### Batch Jobs {{< badge version="professional" >}}JobRunr Pro{{< /badge >}}
- `ConcurrentJobModificationPolicy` has been dropped with no alternative 
- `DashboardDbDataProvider` public constructor now also expects a `BatchJobManager`
- `JobSearchRequest` now requires a list of UUID for awaitingOn
- `StorageProvider.BatchJobWithChildStats` and `StorageProvider.BatchJobChildStats` now also require an awaitingBatchJobChildCount

### Job States {{< badge version="professional" >}}JobRunr Pro{{< /badge >}}
- `AllowedJobStateStateChanges.isIllegalStateChange(...)` now also expected a JobType
- `DeletedState`: the constructor `DeletedState(String reason, JobRunrUserInfo user)` has been removed
- `EnqueuedState`: the constructor `EnqueuedState(String reason, JobRunrUserInfo user)` has been removed

### Miscellaneous Changes
- JobRunr test fixtures now require JDK 17 or higher
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} Remove `CronExpression#create`, use constructor instead
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} Several unused constructor on `RoundRobinDynamicQueuePolicy` had been removed
- {{< badge version="enterprise" >}}JobRunr Pro{{< /badge >}} The Multi-Cluster Dashboard should be upgraded at the same time as the single cluster dashboard (this is not required if they were on 8.7 or higher)
- {{< badge version="professional" >}}JobRunr Pro{{< /badge >}} Removed duplicate `JsonMapperFactory`, use the factory in package `org.jobrunr.utils.mapper`
